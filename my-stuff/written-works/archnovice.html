Arch Linux Novice: Zero to Usable OS

Introduction

This document exists as a sort of "introduction" into the wonderful world of Linux; explaining the most crucial ideas that you will need to understand in order to make the most of your system. Throughout this document, I will try my best to assume no prior knowledge of computers, software and hardware, that the average "normal" person wouldn't have. To follow along with this, you will need the following basic requirements:

  • A bit of experience using common operating systems like Windows

  • An x86_64-(sometimes called x64)-based computer with WiFi support which you want to install Linux on

  • A genuine willingness to learn about how things work on your computer

  • The ability to comprehend and research new, possibly unintuitive information

If you have these few requirements, and you put your mind to the task, you will be able to gain a deep understanding in and experience with Linux. If you do not understand a certain part of what I talk about here, I seriously encourage you to research further and find out more.

By the end of this, you will have installed an operating system called Arch Linux; and hopefully will have learnt a lot along the way. Arch Linux is typically not recommended for beginners with zero Linux experience, as setting it up and getting things to work is substantially "harder" on it than on some other operating systems. I think this is completely the wrong approach to take. Sure, if you just want a usable computer and you aren't a very curious person, that's fair. You just need to get work done and you don't care about how your computer works under the hood or how its underlying structures are set up - but what's the fun in that? By successfully setting up and adapting to something as foreign and different as Arch Linux, you will gain a lot of insight into how you computer works and the different ways it could work, should you desire.

I hope you find this instructional document useful. I have tried to keep it concise while still detailing everything you will need to know.

Preliminary

Parts of a computer

Modern computers are made of many different components, put together in such a way that they can interact with each other in useful ways. The most important components that you need to know about are the following:

CPU

The CPU (Central Processing Unit) is generally considered the "brain" of a computer. When programmers write code for a computer to execute, the CPU is the part which actually does the execution. You can think of it as performing three operations perpetually, while the computer is on: "fetch", "decode", "execute". First, the CPU finds the part of the code that the programmer told it to execute. Then, it figures out what this code means. Finally, it actually performs what it is told: this could be anything from adding and subtracting two numbers and storing the result, to handling the complex logic of videogames. After this, it moves back onto the first step, and fetches the next part of the code. A CPU can do only one or a few things at a time, but it does them really fast

GPU

The GPU (Graphics Processing Unit) works in a somewhat similar way to the CPU. It, too, is used for executing code, but the code it executes is different from the code that the CPU executes. See, unlike the CPU (which can do only a few things but does them very fast), the GPU can do very many things, but it doesn't do them as fast. This makes it suited to different tasks than the CPU. While the CPU is best suited to complex logic and the handling of basic programs, the GPU - due to its highly parallel nature - is best suited to simple, but very repetitive tasks. For example, a videogame calculating how each pixel on the screen should be rendered is a very predictable, repetitive task. You could, theoretically, perform it on a CPU - as it would be able to handle it - but you would get terrible framerate. See, even though the CPU is fast, the sheer number of operations that need to be performed completely overwhelm it. Thus, they are done on the GPU - each "thread of execution" is given only a few tasks individually, but put together, their collective speed allows the GPU to be significantly faster for videogame rendering than the CPU

RAM

RAM (Random Access Memory) can be thought of as the "short term memory", or "working memory" of a computer. Whenever the computer has performed some computation and it needs to store the result for later, it will write to and read from RAM. As well as this, it is important to understand that a CPU cannot run a program which is stored on the disk directly - it must first read this program and put it into RAM, as the CPU in its basic form cannot directly access the disk. The reason people don't use RAM for everything, and your computer still has a disk in it, is that RAM requires continuous power to be supplied to it at regular cycles, or else the data stored on it will be lost. Thus, when you turn off your computer, all information stored in RAM is lost. This is why when you try to shut down your computer without saving a Microsoft Word document, Windows will warn you that you will lose data - because the current, unsaved version of the document is stored in RAM so that the CPU can access it and modify it in the ways you tell it to. Unfortunately, this means that when the RAM no longer recieves any power (i.e. when you shut down your computer), the unsaved version of the document will be lost
